If you want to setup a brand new Registration and need to see all your available options, start with this: http://www.bvcms.com/Page/SampleOnlineRegSettings. 

To see the settings page on your Organization, just hover over the "Other Management" dropdown menu and click on the "Online Registration Settings" item.

A keyword might expect a value to the right of the : (colon). These are the types of values that can be expected and the keywords table is organized by type.

Small Groups and Fees

In the following documentation, there is a pattern where a SmallGroup keyword shows up along with a Fee keyword, all below the description. If no Small Group is specified, the description itself becomes the name of the SmallGroup. The SmallGroup is used to classify the selection the registrant made and is available for filtering and managing your groups within the organization. You can prefix the small groups with a few letters and a colon like SG:Boys. This can help you distinguish the various sets of small groups.

If a Fee applies, that amount, of course, will tally into the total amount of the transaction.

AgeGroups

These don't display on your form. Instead, they serve to classify with a SmallGroup the age that the registrant falls into. You specify an age range (inclusive of both numbers) and if the person falls into that range, they are assigned to that SmallGroup in the Organization.

Example:
AgeGroups:
  	0-12
  		SmallGroup: Children
  		Fee: 5
  	13-19
  		SmallGroup: Youth
  		Fee: 10
  	20-99
  		SmallGroup: Adults
  		Fee: 20

ShirtSizes:

YesNoQuestions

These become a set of two radio buttons for each question. One for Yes and one for No. The registrant must reply to one or the other. This is unlike checkboxes where it is not required to check a box. This way, you can force an answer, whether negative or positive.

YesNoQuestions:
  	Are you a Facebook User?
  		SmallGroup: Facebook
  	Are you a Twitter User?
  		SmallGroup: Twitter

These become a series of labels with a long text box next to each allowing the registrant to answer the question posed in free form text. This information is stored in the extra data area of the MemberDialog box.

Confirmation

The email confirmation can be composed in the email text editor then view the Sourcek, copy the HTML code and paste into this section.

This section starts with the Confirmation keyword and has two required first level indent keywords, Subject and Body. The subject can be plain text to the right of the keyword (with the colon:). You can optionally specify an Html: keyword which is a true/false value. If true, you can use HTML for the Body, if not, you use plain text. Although you can place the body of the message to the right of the Body keyword just like you did with the subject, typically you want a multi-line message with perhaps some HTML codes. Since lines and indentations are significant in this text document, there is a special syntax used to enter the multi-line text. The Body keyword has the value of << (two left pointy brackets). This is the indicator that two special, marker lines will follow. The next line becomes the marker and is not part of the message. In this case below, we used a line of 10 dash characters. But you could use a word or what ever you want. The only requirement is that this same line show up again marking the end the multi-line message.

Confirmation:
	Subject: Your confirmation
	Html: true
	Body: <<
----------
<p><img src="https://cms.bellevue.org/images/bvcms130.png" /></p>
<p> Thank you {first},

<p> We will be in touch soon with more information.</p>
{details}
----------

Instructions

The instructions can also use the Html keyword but typically can all be one liners of text.

There are five messages available. You do not have to use all five. But they show up in the five sections of the login process.

The Login text up when you enter your username and password.

The Find text up when you are entering a First name and a Last name, Birthday, email etc. which are used to locate the record. If it is not found, you can add a new record. So the Find can explain this process a bit more fully if you want.

The Select text shows up after you have logged in with a username and password. You family members will be displayed and you can either select one of them by clicking their name or you can enter another person to find.

The Options text shows up after the record has been found and it is asking for you to filling other information regarding the registration.

Finally the Submit text shows up when you are ready to either complete the registration or to add additional registrants.

Just as with the confirmation email above, you can treat any section as Html and use the multi-line syntax to enter the text.

Instructions:
	Select: Choose a family member
	Login: Please login or use the register without account
	Find: This is where we try to find your record
	Sorry: Click here to get on the waiting list
	Html: true
	Options: << 
----------
<p>Fill this information out <i>please</p>
<p>It will be validated when you submit</p>
----------

Terms

The terms can be simple or a long document of text. This will show up on the payment page and require that you check an approve box before you can continue. And again, it could be a one liner or use the multi-line text in conjunction with the Html keyword.

Terms:
	Body: No lawsuits, OK?
